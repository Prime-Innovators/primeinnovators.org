## Your First Open Source Contribution

Making your first open-source contribution is intimidating. Here's a step-by-step guide based on what we've seen work for hundreds of students.

### Step 1: Find the Right Project

Look for projects with:
- `good-first-issue` or `help-wanted` labels
- Active maintainers (recent comments on PRs)
- A contributing guide (`CONTRIBUTING.md`)
- A welcoming code of conduct

### Step 2: Start Small

The best first contribution isn't code. It's:

1. Fixing a typo in documentation
2. Improving error messages
3. Adding tests for edge cases
4. Updating dependencies

### Step 3: The PR Workflow

```bash
git checkout -b fix/typo-in-readme
# make your change
git add .
git commit -m "docs: fix typo in README"
git push origin fix/typo-in-readme
# open a PR on GitHub
```

### Common Mistakes

- Not reading the contributing guide
- Making huge changes in one PR
- Getting defensive about feedback

Open source is collaborative, not competitive. Every contribution counts.
