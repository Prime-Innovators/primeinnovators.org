## Security at the Edge

Running code on Cloudflare's edge network means your security perimeter is wherever your code runs. Traditional VPN-based approaches don't apply.

### Zero Trust Principles

1. **Verify every request**: No implicit trust, even from "internal" networks
2. **Least privilege access**: Workers only access what they need
3. **Encrypt everything**: Data in transit and at rest

### Our Worker Security Model

```js
// Every Worker request goes through this chain
async function handleRequest(request) {
  const authenticated = await verifyJWT(request);
  const authorized = await checkPermissions(authenticated);
  const sanitized = await validateInput(request);
  return routeRequest(sanitized);
}
```

### D1 Database Isolation

With Cloudflare D1, each tenant's data is isolated at the query level. We use prepared statements and row-level security to ensure that user A never sees user B's data — even if they share the same Worker.

Edge security isn't harder — it's just different.
