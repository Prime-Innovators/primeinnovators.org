## Serverless Databases Are Different

Traditional database design assumes a long-lived connection pool. On Cloudflare Workers, each request is ephemeral. This changes everything.

### D1 Design Principles

**1. Connection per Request**

```sql
-- Every Worker invocation opens a fresh D1 connection
-- Design your queries to be fast and self-contained
SELECT * FROM users WHERE id = ?;
```

**2. Batch Writes**

D1 supports batched statements. Instead of 10 individual inserts, batch them into one round trip.

**3. Index Everything You Query**

Without connection pooling, a full table scan on every request is deadly. We index every column used in `WHERE`, `JOIN`, or `ORDER BY`.

### Our Schema

```sql
CREATE TABLE waitlist (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT UNIQUE NOT NULL,
  role TEXT NOT NULL,
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_waitlist_role ON waitlist(role);
```

### What We'd Do Differently

Start with D1's production limits in mind: 10GB per database, 1000 requests per second. Design for partitioning from day one.
