## RSC and the Static Site

React Server Components (RSC) are reshaping how we think about server-rendered React. But for fully static sites, the calculus is different.

### Why We Skipped Next.js

Next.js is the obvious choice for RSC, but for our use case:

- **We don't need a server**: Cloudflare Pages serves static files
- **Build-time rendering is faster**: No server cold starts
- **Simpler deployment**: Just upload HTML files

### What vite-react-ssg Gives Us

The `vite-react-ssg` plugin provides:

| Feature | Benefit |
|---------|---------|
| SSG with React Router | Familiar routing, static output |
| Dynamic routes | Blog posts as SSG pages |
| Lazy loading | Code-split pages automatically |
| Helmet support | Per-page SEO metadata |

### Tradeoffs

Of course, there are tradeoffs. Without RSC, we can't do streaming or incremental rendering. But for a content site where milliseconds matter, pure static HTML is hard to beat.
